Driving School Management System
Проект представляет собой реляционную базу данных для автоматизации бизнес-процессов автошколы. Включает в себя полный цикл ведения студента: от заключения договора до сдачи внутренних и внешних экзаменов в ГАИ.

О проекте
Данная работа выполнена в рамках лабораторной работы по проектированию баз данных. Цель проекта - создать отказоустойчивую структуру данных, которая минимизирует человеческий фактор при распределении практических занятий и контроле допуска студентов.

Ключевые функции:
Учет филиалов и персонала: Разделение на теоретических преподавателей и инструкторов по вождению.
Управление автопарком: Привязка автомобилей к инструкторам с учетом типа трансмиссии (МКПП/АКПП).
Учебный процесс: Контроль посещаемости лекций и учет наезженных часов (минут) на практических занятиях.
Автоматизация допуска: Триггерная система, которая автоматически меняет статус студента на «Допущен» только при выполнении нормы учебного плана по теории и практике.

Архитектура данных
ER-диаграмма
Система построена на основе детально проработанной ER-модели (сущности, связи, атрибуты), учитывающей специфику обучения (например, соответствие категории прав инструктора курсу студента).

Реляционная схема
Основные сущности системы:
STUDENT — персональные данные, статус обучения и накопленные часы.
COURSE — параметры программ обучения (Категория, часы, тип КПП).
INSTRUCTOR / VEHICLE — учет ресурсов для практических занятий.
PRACTICE_LESSON — журнал вождения с фиксацией километража и времени.
EXAM / STUDENT_EXAM — учет результатов внутренних и государственных испытаний.

Реализация и бизнес-логика
В проекте реализованы сложные ограничения целостности через механизмы Triggers и Constraints:
Валидация ресурсов: Триггер trg_practice_attendance_course_fit блокирует назначение автомобиля на занятие, если тип его трансмиссии не совпадает с выбранным курсом студента.
Контроль компетенций: Триггер trg_practice_instructor_category_fit проверяет, совпадает ли категория прав инструктора с категорией курса, на который зачислен ученик.
Автоматический расчет прогресса: После каждого занятия триггер trg_tatt_add_minutes обновляет счетчик часов в профиле студента и проверяет условия для перевода в статус «Допущен».
