# Driving School Management System (Database Design)

## Обзор проекта
Проект представляет собой реляционную базу данных для автоматизации ключевых бизнес-процессов автошколы. Система охватывает полный жизненный цикл обучения студента: от первичной регистрации и заключения договора до фиксации результатов внутренних и государственных экзаменов в ГАИ.

Данная работа направлена на создание отказоустойчивой структуры данных, минимизирующей влияние человеческого фактора при распределении ресурсов и контроле учебного прогресса.

## Ключевые функции
* **Учет ресурсов и персонала:** Разделение штата на преподавателей теории и инструкторов по вождению с привязкой к конкретным филиалам.
* **Управление автопарком:** Учет транспортных средств с фиксацией типа трансмиссии (МКПП/АКПП) и их закрепление за инструкторами.
* **Мониторинг учебного процесса:** Автоматизированный учет посещаемости теоретических лекций и накопления практических часов.
* **Контроль допуска:** Программная проверка выполнения учебного плана для смены статуса студента на «Допущен».

## Архитектура данных

### ER-диаграмма
В основе системы лежит детально проработанная ER-модель, описывающая сущности, их атрибуты и типы связей. Особое внимание уделено логическим ограничениям (например, соответствие категории прав инструктора программе обучения студента).

### Реляционная схема
Основные сущности базы данных:
* **STUDENT** — персональные данные, накопленные часы теории/практики и статус допуска.
* **COURSE** — параметры учебных программ (категория, норма часов, тип КПП).
* **INSTRUCTOR / VEHICLE** — реестр кадровых и технических ресурсов.
* **PRACTICE_LESSON** — журнал практических занятий с фиксацией пробега и длительности.
* **EXAM / STUDENT_EXAM** — история попыток и результаты аттестации.

## Реализация бизнес-логики
Целостность данных и автоматизация проверок реализованы на уровне СУБД с помощью механизмов Constraints и Triggers:

1. **Валидация ресурсов (`trg_practice_attendance_course_fit`):** Блокирует назначение автомобиля на практическое занятие, если тип его трансмиссии не соответствует выбранному курсу студента.
2. **Контроль компетенций (`trg_practice_instructor_category_fit`):** Проверяет соответствие квалификационной категории инструктора категории обучения студента.
3. **Автоматический расчет прогресса (`trg_tatt_add_minutes`):** После внесения данных о посещении занятия триггер пересчитывает общую сумму учебных часов студента и автоматически обновляет статус допуска при достижении целевых показателей.
